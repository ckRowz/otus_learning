Пользовательские сценарии:

Есть клиент
И Есть виртуальный ассистент (VA)
И Есть Корпоративное приложение для сопровождения нефтегазодобычи (IS)
К Которому VA непосредственно относится
Кодга клиент вводит запрос в поле ввода в интерфейсе ассистента 
Тогда клиент видит ответ в чате

1. Запрос инструкции
Когда клиент вводит запрос вида "как/где сделать что-либо в IS продукте"
Тогда VA отдает ответ в виде инструкции (html/pdf) с коротким тесктом, поясняющим содержание

2. Запрос конкретных значений из IS
Когда клиент запрашивает конкретные значения показателей IS
И Если ответ представляет числовое значение
Тогда VA отправляет это значение в чат
Если ответ представляет табличные данные
Тогда VA отправляет сгенерированный файл Excel в чат
Если ответ представляет растровые изображения и готовые отчеты
Тогда VA отправляеет сгенерированный файл Pdf в чат

3. Реакции на ответы VA
Когда клиент получает ответ от VA
Тогда он может отреагировать на ответ
И Если клиента удовлетворяет ответ
Тогда он может поставить положительную реакцию
Если клиента не удовлетворяет ответ 
Тогда он может поставить негативную реакцию
Если клиент обладает правами эксперта и его не удовлетворяет ответ
Тогда он может отредактировать ответ

Сервисы, выделенные методом функционального моделирования:
1. Сервис диспетчер (Dispatcher)
Регистрация входящих сообщений
Обращение к сервису поиска команды
Перенаправление на другие сервисы

2. Сервис для определения цели запроса по его содержанию (CommandResolver)
Определение целевого сервиса для обработки запроса по входящему сообщению

3. Сервис инструкций (InstructionService)
Найти и вернуть инструкцию по тексту запроса

4. Сервис данных системы (DataService)
Найти в системе и предоставить ответ в нужной форме

5. Сервис генерации отчетов (ReportService)
Сгенерировать отчет в укзанном формате

6. Сервис регистрации фидбэка клиента (FeedbackService)
Зарегистрировать реакцию клиента на ответ для корректировки ИИ моделей и алгоритмов
